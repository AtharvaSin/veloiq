03 — Database Schema

Nine tables, five pipeline stages

Tables are colour-coded by their role in the 4-stage pipeline. The audit_log table is cross-cutting and append-only — the application role has UPDATE and DELETE revoked at the PostgreSQL layer, making audit entries immutable once committed.

Entity-relationship diagram — 9 tables, colour-coded by pipeline stage
Immutability guarantee

Migration 001 executes REVOKE UPDATE, DELETE ON audit_log FROM veloiq after table creation. The application database role physically cannot mutate audit entries after commit. This is the architectural cornerstone for TÜV non-repudiation — every mutation is provable post-hoc.

08 — Audit Enforcement

One transaction, two rows

Every mutation writes two rows atomically: the entity change and its audit entry. If the audit write fails, the entity change rolls back. The application database role cannot UPDATE or DELETE audit entries post-commit.

Audit sequence — atomic mutation + audit write

Reference implementation

Extract from backend/app/services/certificates_service.py:24-59 — the canonical pattern every mutation service follows:

def create_certificate(
    db: Session,
    payload: CertificateCreate,
    actor: str,
) -> Certificate:
    """Create a Certificate and write a paired audit_log entry."""
    cert = Certificate(
        id=uuid4(),
        certificate_number=payload.certificate_number,
        customer_id=payload.customer_id,
        product_description=payload.product_description,
        status=payload.status,
        issue_date=payload.issue_date,
        expiry_date=payload.expiry_date,
    )
    db.add(cert)
    db.flush()

    write_audit_entry(
        db,
        entity_type="certificate",
        entity_id=cert.id,
        action="created",
        actor=actor,
        details={
            "certificate_number": cert.certificate_number,
            "customer_id": str(cert.customer_id),
            "status": cert.status,
            "issue_date": cert.issue_date.isoformat(),
            "expiry_date": cert.expiry_date.isoformat(),
        },
    )

    db.commit()
    db.refresh(cert)
    return cert
Atomicity

Both INSERT (entity) and INSERT (audit_log) execute in the same db.commit(). If either fails, the transaction rolls back and no rows are written. The pairing is guaranteed at the database level, not just at application level.

Immutability

Migration 001 executes REVOKE UPDATE, DELETE ON audit_log FROM veloiq. The application role owns audit entries only in their create phase. Once committed, no SQL executed by the application can alter them.
